用于Markdown的图床

 2020.12.10  tech  

昨天学画图的时候发现一个问题，图片放哪儿？？放本地的话笔记要转移就只能连带图片一起打包或者把 Markdown 导出为PDF，挺麻烦的。所以要把图片放在服务器端，有网就可以访问，这样就方便多了，刚好我的一些笔记会传到博客上，完美契合。这里我有几个需求：

  • 可以将图片分类存放。分不同文件夹就可以实现了。
  • 上传要方便，最好是即插即传，写笔记插入图片的时候直接就上传到服务器端最好。
  • 访问速度要快，非本地的话加载图片最好别有明显延迟，影响阅读。

网上检索了一番，最后选择用 picGo + github 做图床。因为我用 Typora 写笔记，刚好最近更新加入了图片上传功能，直接调用 PicGo 将插入的本地图片上传到github 仓库，获取图片链接，显示，理论上非常流畅。至于github访问速度的问题，网上检索之后选择用 jsDelivr 加速。环境已经配置好了，现在体验一番。

首先，我有一张本地图片，路径为：
